# Design Tokens

Design tokens are the visual design atoms of the Brain Researcher design system. They define the consistent visual properties used throughout the application.

## Colors

Our color system is designed for accessibility and scientific data visualization, with high contrast ratios and colorblind-friendly palettes.

### Primary Colors
- **Primary**: `hsl(221.2 83.2% 53.3%)` - Brand blue for primary actions
- **Primary Foreground**: `hsl(210 40% 98%)` - Text on primary backgrounds
- **Secondary**: `hsl(210 40% 96%)` - Secondary actions and backgrounds
- **Secondary Foreground**: `hsl(222.2 84% 4.9%)` - Text on secondary backgrounds

### Semantic Colors
- **Destructive**: `hsl(0 84.2% 60.2%)` - Error states and dangerous actions
- **Destructive Foreground**: `hsl(210 40% 98%)` - Text on destructive backgrounds
- **Success**: `hsl(142.1 76.2% 36.3%)` - Success states and confirmations
- **Warning**: `hsl(47.9 95.8% 53.1%)` - Warning states and cautions

### Neutral Colors
- **Background**: `hsl(0 0% 100%)` / `hsl(222.2 84% 4.9%)` (dark)
- **Foreground**: `hsl(222.2 84% 4.9%)` / `hsl(210 40% 98%)` (dark)
- **Muted**: `hsl(210 40% 96%)` / `hsl(217.2 32.6% 17.5%)` (dark)
- **Muted Foreground**: `hsl(215.4 16.3% 46.9%)` / `hsl(215 20.2% 65.1%)` (dark)

### Borders
- **Border**: `hsl(214.3 31.8% 91.4%)` / `hsl(217.2 32.6% 17.5%)` (dark)
- **Input**: `hsl(214.3 31.8% 91.4%)` / `hsl(217.2 32.6% 17.5%)` (dark)
- **Ring**: `hsl(221.2 83.2% 53.3%)` - Focus rings

## Scientific Data Colors

### Brain Activation Maps
- **Positive**: `hsl(0 100% 50%)` to `hsl(60 100% 50%)` - Red to yellow gradient
- **Negative**: `hsl(240 100% 50%)` to `hsl(180 100% 50%)` - Blue to cyan gradient

### Statistical Significance
- **p < 0.05**: `hsl(142.1 76.2% 36.3%)` - Significant (green)
- **p < 0.01**: `hsl(142.1 76.2% 26.3%)` - Highly significant (dark green)
- **p ≥ 0.05**: `hsl(215.4 16.3% 46.9%)` - Non-significant (gray)

## Usage Guidelines

### Do's
- Use tokens consistently across components
- Test color combinations for accessibility
- Maintain hierarchy with typography scales
- Apply consistent spacing patterns

### Don'ts  
- Don't hardcode colors or spacing values
- Don't use colors that fail accessibility standards
- Don't mix different font families unnecessarily
- Don't break the spacing scale for custom values
